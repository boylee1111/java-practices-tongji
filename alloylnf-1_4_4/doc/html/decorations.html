Using Look&Feel Frame Decorations

Basics

Since version 1.2 the Alloy Look&Feel supports outer Frame decorations that match the look of the Alloy Look&Feel. This functionality is only available when used with JRE/JDK versions 1.4 or higher.

There are two ways to enable the look&feel frame decorations:

  1. Call the methods JFrame.setDefaultLookAndFeelDecorated(true) and JDialog.setDefaultLookAndFeelDecorated(true) in your application. This will result in each new JFrame and JDialog having the decorations provided by the Alloy Look&Feel. Note that both methods are only available since JRE/JDK 1.4. With JRE/JDK 1.3 and 1.3.1 a NoSuchMethodException will be thrown.
  2. Set the property "alloy.isLookAndFeelFrameDecoration" to "true". Look at the section about properties to learn how to set a property in the Alloy Look&Feel. Using a property for setting the look&feel frame decoration has the advantage that you are independant of the JRE/JDK version. For version 1.3 and 1.3.1 the property will just be ignored.

Notes

The look&feel frame decorations has certain limitations that you should know about. Read this first before using the look&feel frame decorations in your application to avoid surprising behaviour.

  • The look&feel frame decorations finally honor the minimum size of a frame, which is a very nice feature, but it can also result in strange effects in some cases. The reason for this is that one of the parameters used for calculating the minimum size of a frame is the minimum size of the frame's content pane. In some cases the minimum size of a content pane can differ depending on its current size, which leads to different minimum sizes of a frame for different resizing operations. A simple workaround for this problem is to set the minimum size of  the content pane to a fixed value, which can look like this:

    Container contentPane = myFrame.getContentPane();
    if (    contentPane instanceof JComponent) {
      ((JComponent)     contentPane).setMinimumSize(new Dimension(100, 100));
    }

  • Please note that in order to remove all decorations from a frame, you will have to call myFrame.getRootPane().setWindowDecorationStyle(JRootPane.NONE) in addition to myFrame.setUndecorated(false). The method setUndecorated(false) only removes the native decorations from the frame, but is does not affect the decoration style provided by the look and feel.
  • Unlike native decorations the system menu of the look&feel frame decoration does not alloy choosing an input method (e.g. for Japanese text input). This affects only very few applications, but you should be aware of it. Currently we are investigating if there is a way to provide this functionality with look&Feel frame decorations.
  • If you prefer to use the "classic" style of frame buttons that was used until Alloy 1.3.1 you can set this via setting the property "alloy.frameButtonStyle" to "classic". Details can be found in the section about properties.
