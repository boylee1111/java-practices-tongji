Customizing Fonts

To customize the fonts that are used by the Alloy Look&Feels for displaying text on components the interface com.incors.plaf.alloy.AlloyFontTheme is provided.  It defines methods for retrieving the fonts that should be used by the Alloy Look&Feel. The API for the interface com.incors.plaf.alloy.AlloyFontTheme can be found here.

Each theme for the Alloy Look&Feel should provide a constructor that takes an instance of com.incors.plaf.alloy.AlloyFontTheme as an argument. So if you have an instance of a class that implements the interface com.incors.plaf.alloy.AlloyFontTheme, you can use it like shown below (with the Glass theme).

/* myFontTheme is an instance of a class that implements AlloyFontTheme */
com.incors.plaf.alloy.themes.glass.GlassTheme glassTheme = new com.incors.plaf.alloy.themes.glass.GlassTheme(myFontTheme);

The Class CustomFontTheme

One class that implements this interface is com.incors.plaf.alloy.themes.CustomFontTheme, which was created to make the customization of fonts as easy as possible. Its API can be found here. When creating an instance of com.incors.plaf.alloy.themes.CustomFontTheme you can propose an array of font names and the font sizes that should be used by the look and feel. The object will then try to find one of the proposed fonts on the system. If no font is found, the system font Dialog will be used as a fallback font.

This example below shows how the class com.incors.plaf.alloy.themes.CustomFontTheme can be used with the Acid theme (usage with other themes is basically the same):

com.incors.plaf.alloy.themes.CustomFontTheme myFontTheme = new com.incors.plaf.alloy.themes.CustomFontTheme(new String[] {"Tahoma", "Verdana", "Helvetica"}, 12, 10);
com.incors.plaf.alloy.themes.acid.AcidTheme acidTheme = new com.incors.plaf.alloy.themes.acid.AcidTheme(myFontTheme);
com.incors.plaf.alloy.AlloyLookAndFeel alloyLnF = new com.incors.plaf.alloy.AlloyLookAndFeel(acidTheme);

Note: If you want to use a serif font (like Times New Roman) for the Alloy Look&Feel it might be a good idea to add Serif to the array of proposed font names. This font is a Java™ system font and always available, so it is a good fallback solution for this case.

Custom Implementations of AlloyFontTheme

To create your own implementation of com.incors.plaf.alloy.AlloyFontTheme you should first take a look at its API, which can be found here. The class HelveticaFontTheme is an example that shows how an implementation of the interface com.incors.plaf.alloy.AlloyFontTheme could look like. Note that this class is only for demonstration. You will not find it in the Alloy Look&Feel distribution.

import java.awt.Font;
import javax.swing.plaf.FontUIResource;
import com.incors.plaf.alloy.AlloyFontTheme;

public class HelveticaFontTheme {
  private FontUIResource controlTextFont;
  private FontUIResource systemTextFont;
  private FontUIResource menuTextFont;
  private FontUIResource userTextFont;
  private FontUIResource windowTitleFont;
  private FontUIResource subTextFont;
 
  public FontUIResource getControlTextFont() {
    if (controlTextFont == null) {
      controlTextFont = new FontUIResource(new Font("Tahoma", Font.PLAIN, 12));
    }
    return controlTextFont;
  }

  public FontUIResource getSystemTextFont() {
    if (systemTextFont == null) {
      systemTextFont = new FontUIResource(new Font("Tahoma", Font.PLAIN, 12));
    }
    return systemTextFont;
  }

  public FontUIResource getMenuTextFont() {
    if (menuTextFont == null) {
      menuTextFont = new FontUIResource(new Font("Tahoma", Font.BOLD, 12));
    }
    return menuTextFont;
  }

  public FontUIResource getUserTextFont() {
    if (userTextFont == null) {
      userTextFont = new FontUIResource(new Font("Tahoma", Font.PLAIN, 12));
    }
    return userTextFont;
  }

  public FontUIResource getWindowTitleFont() {
    if (windowTitleFont == null) {
      windowTitleFont = new FontUIResource(new Font("Tahoma", Font.BOLD, 12));
    }
    return windowTitleFont;
  }

  public FontUIResource getSubTextFont() {
    if (subTextFont == null) {
      subTextFont = new FontUIResource(new Font("Tahoma", Font.PLAIN, 10));
    }
    return subTextFont;
  }
 
}

Things to keep in mind

Some hints for implementing custom font themes:

  • Keep in mind that on different systems different fonts are installed.
  • Most fonts can only display Latin characters. Keep this in mind if you are planning to provide localized versions of your software that have to display other characters, e.g. Cyrillic or Chinese characters.
