Setting the Alloy Look&Feel as a JRE's Default Look&Feel

Making the Alloy Look&Feel the default look and feel of your Java™ Runtime Environment is quite easy.

First you have to put the file alloy.jar into the folder /lib/ext of your JRE. Then you have to create a properties file with the name alloy.properties, which contains at least the license code property. The file alloy.properties is a simple text file which contains this one line of text:

alloy.licenseCode=[license Code]

where [license Code] must be replaced by your license code. This file must be put into a jar archive. You can use the jar utility that comes with the JDK or simply create a zip archive with your favorite zip tool, add the properties file to the zip archive and replace the .zip suffix of the archive with .jar. This jar has to be put into the folder /lib/ext of your JRE, too. It does not matter which name you give to the jar archive that contains the alloy properties file, as long as it ends with .jar . To make it clear what is inside the jar we usually call it alloyProperties.jar.

The last thing you need to do is to create another properties file, with the name swing.properties . This file is also a simple text file which contains the following text:

swing.installedlafs = motif,windows,metal,mac,alloy
swing.installedlaf.motif.name = CDE/Motif
swing.installedlaf.motif.class = com.sun.java.swing.plaf.motif.MotifLookAndFeel
swing.installedlaf.windows.name = Windows
swing.installedlaf.windows.class = com.sun.java.swing.plaf.windows.WindowsLookAndFeel
swing.installedlaf.metal.name = Metal
swing.installedlaf.metal.class = javax.swing.plaf.metal.MetalLookAndFeel
swing.installedlaf.mac.name = Mac
swing.installedlaf.mac.class = com.sun.java.swing.plaf.mac.MacLookAndFeel
swing.installedlaf.alloy.name = Alloy
swing.installedlaf.alloy.class = com.incors.plaf.alloy.AlloyLookAndFeel
swing.defaultlaf=com.incors.plaf.alloy.AlloyLookAndFeel

The file swing.properties has to be put into the folder /lib of your JRE.

If you followed these instructions, Swing applications that run on your JRE should now use the Alloy Look&Feel as their default look and feel.

Using the Alloy Look&Feel with IntelliJ IDEA

To use the Alloy Look&Feel with IntelliJ IDEA you will first have to find the JRE that is being used by IntelliJ IDEA on your system. By default the used JRE can be found in the subfolder /jre of the IntelliJ IDEA program folder. Add the Alloy Look&Feel to that JRE as described in the section "Setting the Alloy Look&Feel as an JRE's Default Look&Feel". Then start IDEA and select Alloy from the combo box in Options->IDE-Settings->Appearance->Look And Feel.

Using the Alloy Look&Feel with Netbeans

To use the the Alloy Look&Feel with netbeans all you have to do is to follow these two steps:
  1. Put the file alloy.jar into the lib/ext/ folder of netbeans.
  2. Add the following to the text of the file ide.cfg, which is located in the bin/ folder of netbeans (if the file ide.cfg does not exist, simply create an empty text file with that name).

    -ui com.incors.plaf.alloy.AlloyLookAndFeel -J-Dalloy.licenseCode=[license code]

    The part [license code] must be replaced by your license code. Please be aware that the license code cannot be put in quotation marks, because netbeans does not recognize them properly.
That all you have to do. Netbeans should now use the Alloy Look&Feel as the default look and feel.

Using the Alloy Look&Feel with JBuilder

To use the Alloy Look&Feel with JBuilder you will first have to find the JRE that is being used by JBuilder on your computer. By default the used JRE is located in the JBuilder program folder (there is no /jre folder, only /lib and /bin). Add the Alloy Look&Feel to that JRE as described in the section "Setting the Alloy Look&Feel as an JRE's Default Look&Feel". Then start JBuilder and set the look and feel in Tools->IDE Options->Browser->Look and Feel.
