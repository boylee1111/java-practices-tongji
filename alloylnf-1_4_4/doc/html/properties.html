Properties

Since version 1.1 the Alloy Look&Feel allows the use of properties. Currently properties can be used for setting the license code, the default theme, the frame decoration style, and to indicate if the application is running in a sandbox. We intend to add more properties for customization of the Alloy Look&Feel in future.

Supported Properties

Currently supported properties are:

- must be set before instantiating the Alloy Look&Feel

alloy.isSandbox
 indicates if the applications is running in a Java WebStart sandbox

true for applications that are running in a Java WebStart sandbox
false (default value) for normal applications and for trusted Java WebStart applications
alloy.isLookAndFeelFrameDecoration
 - indicates if  decorations that match the Alloy Look&Feel should be used for frames and dialogs
- will be ignored for JRE versions older than 1.4
- must be set before instantiating the Alloy Look&Feel
 true use decorations provided by the Alloy Look&Feel for frames and dialogs
false (default value) use decorations provided by the operating system for frames and dialogs
alloy.isPopupShadowEffectEnabled
 -indicates if menus, popup menus and the combo box popup should drop a small shadow
- must be set before instantiating the Alloy Look&Feel
 true (default value) drop shadows

- must be set before instantiating the Alloy Look&Feel

- must be set before instantiating the Alloy Look&Feel

Setting Properties

Properties can be set by using the static method AlloyLookAndFeel.setProperty(String key, String value).  The following example shows how the license code property can be set with this method:

AlloyLookAndFeel.setProperty("alloy.licenseCode", "x#myCompany#dv34dw#s2wp8");
AlloyLookAndFeel alloyLnf = new AlloyLookAndFeel();
UIManager.setLookAndFeel(alloyLnF);

Note that some properties must be set before an instance of the class AlloyLookAndFeel is created.

Alternative Ways of Setting Properties

You can also set the properties from outside of the software. One way is to pass them as parameters at the commandline when starting the application, the same way as system properties aare set at startup, by simply using -D[property name]=[property value].

When starting the application MyApp on Windows passing the theme parameter can look like this:

java.exe -Dalloy.theme="bedouin" com.myCompany.MyApp

Another alternative way of setting properties is to use a properties-file. You will have to create a simple text file and name it alloy.properties. In this text file you write the properties. Each line must contain one key/value pair. Key and value are separated by an equals sign. The text of a properties file can look like this:

alloy.licenseCode=3#myCompany#dshj2e#87we2j
alloy.theme=bedouin
alloy.isLookAndFeelFrameDecoration=true
alloy.isSandbox=true

The properties file has to be put somewhere in the class path of your application, where the class AlloyLookAndFeel can access it.
